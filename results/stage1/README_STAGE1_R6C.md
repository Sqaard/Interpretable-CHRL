# Stage 1 R6c Hidden-State Codes

Model:
`R6c_root_K20_stock_K5_PD_mild_slice_group_riskaware_top8_sell12_rotation_internaldays_v1`

This package mirrors Joseph's Stage 1 handoff shape:

- `train_codes.parquet`: `date`, `code_id`, `valid`
- `code_summary.csv`: per-code return/VIX/regime summary
- `stage1_metrics.csv`: K sweep metrics with Joseph-compatible columns

Additional files for the more interpretable R6c teacher:

- `behavior_log_daily.parquet`: daily code assignments joined to R6c logs
- `code_behavior_summary.csv`: per-code means for cash/risk/confidence/top-k/group logs
- `r6c_stage1_hidden_state_package.npz`: hidden states, action params, selected behavior arrays

Important alignment detail:
R6c uses K-window execution. One PPO policy call may generate several internal
trading days. The extracted policy hidden state is therefore broadcast to all
daily rows generated by that macro-step. This is intentional: Stage 1 codes
represent the policy state that controlled that K-window segment.


Market context note:
`mean_vix` and `mean_regime_p1` in `code_summary.csv` are computed from the
model-ready normalized features logged by R6c. Use `code_behavior_summary.csv`
and `behavior_log_daily.parquet` for detailed calibrated comparisons across
cash/risk, confidence, group, and Top-K behavior.

Selected K: `8`
Window length: `17` centered trading days.
Boundary rows with incomplete centered windows are marked `valid == False`.
